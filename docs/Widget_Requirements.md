# SlowDash 卡片 (Widget) 设计规范

## 1. 设计目标
* 保持极简、静谧、墨水屏友好的视觉风格。
* 数据优先，避免冗余标题、装饰性图标或复杂边框。
* 针对 1072x1448 分辨率的 Kindle 显示器进行像素级完美 (Pixel-Perfect) 的排版，获得清晰、高对比度的阅读体验。
* 摒弃动态流式网格，采用严格锁死的 **5x7 固定网格 (Fixed Grid)**，追求极致的版面秩序感。
* 支持“一个 Widget 多种形态”与“预览模式下展示多变体”的设计需求。

## 2. 核心术语
* **Widget / 卡片**：SlowDash 中的最小展示单元。
* **size**：卡片在网格中的绝对占位，严格遵循 5 列标准。使用 `WxH` 表示，例如 `1x1`、`1x2`、`4x2`、`5x1`。
* **type**：同一 Widget 的不同展示形态，如 `current`、`forecast`、`recently_added`。
* **Gallery Mode**：后台预览所有 Widget 可用变体的模式，用于展示可配置的尺寸+类型组合，不等同于实际终端布局。
* **Background Mode**：仅限少数视觉类 Widget 的全屏或大尺寸背景展示，通常仅允许一个背景 Widget。

## 3. 5x7 固定网格系统 (Grid System)
SlowDash 针对 Kindle (1072x1448) 采用严格锁死的 5x7 布局。服务端渲染时，CSS Grid 应遵循以下物理推演：
* **屏幕总尺寸**：1072px (宽) x 1448px (高)
* **左右外边距 (Margin-X)**：36px（两侧共 72px，核心内容区宽度恒定为 1000px）
* **上下外边距 (Margin-Y)**：20px
* **间距 (Gutter)**：20px（列间距与行间距统一）
* **基础网格单元 (1x1)**：**184px x 184px**

## 4. 墨水屏 (E-ink) 渲染与视觉约束
受限于墨水屏的 16 级灰度，在服务端生成静态 HTML 并交由 Puppeteer 截图时，必须遵守以下前端约束：
* **线条与边框**：避免细微灰度线条（极易发虚）。强制使用纯黑（如 `border-black`），或将线宽加粗至 `2px` 以确保边缘犀利。
* **字重控制 (Font Weight)**：禁止使用 `font-light` 或过细的字体。常规标签文本起步使用 `font-medium`；核心大数字必须使用粗体无衬线字体 (`font-bold` 或 `font-black`)，以强化视觉抓力。
* **高反差与防锯齿**：对于使用纯黑背景、白色文字的反色卡片，应确保区块边缘对齐整数像素，防止与白底交界处产生次像素级锯齿。
* **图像预处理**：包含丰富色彩的图片（如电影海报），应在服务端提前进行高对比度去色、增加锐化或网点化 (Halftone) 处理，保证在墨水屏上的阶调清晰，不宜过度依赖底层默认抖动算法。

## 6. 背景模式说明
* 背景模式仅用于视觉类 Widget，例如 Plex、Unsplash 等。
* 只允许一个背景 Widget 覆盖全屏，其余 Widget 在其上方正常显示。
* 如果某组件当前没有专门 `background` type，应描述为“可扩展为背景模式”，而非“已支持”。
